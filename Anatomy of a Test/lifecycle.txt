ลำดับการทำงานของ k6 Lifecycle

    Initialization Phase:
        รันโค้ดที่อยู่นอก setup, default, และ teardown
        ใช้สำหรับการตั้งค่าต่าง ๆ และการเตรียมข้อมูลเบื้องต้น
    Setup Phase (ถ้ามี):
        รันครั้งเดียวเพื่อเตรียมข้อมูลหรือสถานะเริ่มต้นสำหรับการทดสอบ
    Execution Phase:
        รันฟังก์ชัน default ซ้ำ ๆ ตามจำนวน Virtual Users และ Duration ที่กำหนด
    Teardown Phase (ถ้ามี):
        รันครั้งเดียวหลังจากการทดสอบเสร็จสิ้น เพื่อทำความสะอาดระบบหรือปิดการทำงาน

สรุป

Lifecycle ใน k6 ช่วยจัดการลำดับการทำงานของสคริปต์ทดสอบให้มีโครงสร้างที่ชัดเจนและง่ายต่อการจัดการ โดยแบ่งออกเป็น 4 ขั้นตอนหลัก:

    Initialization Phase - เตรียมข้อมูลและตัวแปรเริ่มต้น
    Setup Phase - จัดการข้อมูลเริ่มต้นสำหรับการทดสอบ (Optional)
    Execution Phase - การรันคำขอหลักตามพฤติกรรมที่ต้องการ
    Teardown Phase - ทำความสะอาดระบบหรือปิดการทำงาน (Optional)
